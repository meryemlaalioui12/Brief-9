📊 Marketing Campaign Analysis & Customer Insights
📌 Project Overview

This project focuses on analyzing customer data to evaluate the effectiveness of marketing campaigns and identify the most purchased products.

As a junior data analyst, the goal is to transform raw data into actionable business insights using Python, Power Query, and Power BI.

🎯 Objectives
Perform Exploratory Data Analysis (EDA) to understand customer behavior.
Conduct statistical tests to measure campaign impact.
Clean and transform data using Power Query.
Build a data model in Power BI.
Create an interactive dashboard to monitor campaign performance.
📂 Dataset Description

The dataset includes:

👤 Customer demographics (age, income, country, etc.)
🛒 Transactions (purchases, spending)
📢 Campaign data (accepted/not accepted)
🧾 Products purchased
🌐 Purchase platforms (Web, Store, Catalogue)
⚙️ Tools & Technologies
Python (Pandas, NumPy, Matplotlib, Seaborn) → Data analysis & visualization
Power Query → Data cleaning & transformation
Power BI → Data modeling & dashboard creation
🚀 Project Workflow
🔹 Step 1: Exploratory Data Analysis (EDA)
Load dataset using pandas
Identify variable types:
Continuous variables (e.g., income, spending)
Categorical variables (e.g., country, campaign acceptance)
Perform descriptive statistics:
Mean, median, min, max, standard deviation
Data visualization:
📊 Histograms & boxplots → Sales distribution
📊 Bar charts → Campaign acceptance
Data quality checks:
Missing values
Duplicates
Outliers

✅ Outcome: Initial insights about customer behavior and campaign trends.

🔹 Step 2: Statistical Analysis
Evaluate campaign impact on sales:
T-test / ANOVA → Compare average spending
Chi-square test → Relationship between categorical variables
Correlation analysis:
Identify relationships between demographics and purchases
Interpret p-values:
Determine statistically significant factors

✅ Outcome: Identify which campaigns influence customer purchases.

🔹 Step 3: Data Transformation (Power Query)
Pivot / Unpivot operations:
Campaigns → Single column (accepted campaigns)
Products → Consolidated product sales
Platforms → Web / Store / Catalogue
Data cleaning:
Rename columns
Standardize formats
Ensure relationships:
Customer ID → Campaigns / Products / Platforms

✅ Outcome: Clean, structured dataset ready for modeling.

🔹 Step 4: Data Modeling (Power BI)
Create relationships using:
Customer ID as primary key
Define relationships between:
Customers
Campaigns
Products
Platforms
Create DAX measures:
Total Sales
Sales by Campaign
Sales by Product
Number of customers per campaign

✅ Outcome: Optimized data model for analysis.

🔹 Step 5: Dashboard – Campaign Performance

Interactive Power BI dashboard including:

📊 Number of customers per campaign
💰 Revenue by campaign and product
📦 Product sales distribution per campaign (stacked bar chart)
🌐 Purchase distribution by platform
Filters (Slicers):
Campaign
Product
Country
Platform

✅ Outcome: Business-ready dashboard for decision-making.

📈 Key Insights (Example)
Identify top-performing campaigns
Detect most purchased products
Understand customer preferences by platform
Discover demographic patterns affecting purchases

This project helps:

Support data-driven decision making
